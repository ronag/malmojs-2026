Scheduling & Priorities

@nxtedition/scheduler

The Problem

  • Node.js has a single event loop; all work competes equally
  • A background task can starve a latency-sensitive request
  • No built-in mechanism for task priorities
  • libuv thread pool has a single FIFO queue; high-priority tasks get queued behind low-priority ones

Real-World: SMB vs SSD

// Without scheduling: slow SMB reads block fast SSD reads
// SMB: ~50-500 ms per request, holds a thread pool slot the entire time
// SSD: ~0.1 ms per request, but stuck waiting behind SMB

// 4 concurrent SMB reads fill the thread pool
// → SSD reads queue up, adding 500 ms+ latency to fast ops

// Solution: deprioritize and limit concurrency of slow paths
const scheduler = new Scheduler({ concurrency: 4 })

// Fast path: SSD reads don't need much concurrency (they're fast!)
// but they need quick access to a thread pool slot
await scheduler.run(() => fs.promises.readFile(ssdPath), 'high')

const smbScheduler = scheduler.child({ concurrency: 3 })
// Slow path: SMB reads get low priority + limited concurrency
// Only allow 3 concurrent SMB request → leaves 1 slots for fast work
await smbScheduler.run(() => fs.promises.readFile(smbPath), 'low')
Leave a fast track: limit slow I/O concurrency so fast requests always have thread pool slots available.

Priority-Based Scheduling

import { Scheduler } from '@nxtedition/scheduler'

const scheduler = new Scheduler({ concurrency: 4 })

// High priority, latency-sensitive work
await scheduler.run(() => handleApiRequest(req), 'high')

// Normal priority, standard work
await scheduler.run(() => processMessage(msg), 'normal')

// Low priority, background tasks
await scheduler.run(() => generateReport(), 'low')

// 7 priority levels:
// highest (3) → higher (2) → high (1) → normal (0)
// → low (-1) → lower (-2) → lowest (-3)

Cross-Worker Scheduling

// Main thread: create shared state
const UV_POOL = parseInt(process.env.UV_THREADPOOL_SIZE || '4', 10)
const sharedState = Scheduler.makeSharedState(UV_POOL)

for (let i = 0; i < 4; i++) {
  new Worker('./http-worker.js', { workerData: { sharedState } })
}

// Worker: all workers share the concurrency limit
const scheduler = new Scheduler(workerData.sharedState)
Without coordination, 4 workers each enqueue into the same FIFO thread pool and high-priority tasks get stuck behind low-priority ones.
